# Export Feature Guide

Last updated: 2026-02-14

## Overview

Trispr Flow allows you to export your transcripts in multiple formats for archival, sharing, or integration with other tools. This guide explains how to use the export feature and what each format provides.

---

## How to Export Transcripts

### Quick Start

1. Open Trispr Flow and ensure you have some transcript history
2. Navigate to the **Output** panel
3. Select your desired export format from the dropdown (TXT, Markdown, or JSON)
4. Click the **Export** button
5. Choose a save location in the file dialog
6. Done! Your transcript is saved

### Export Formats

Trispr Flow supports three export formats, each optimized for different use cases:

| Format | Best For | File Size | Readability |
|--------|----------|-----------|-------------|
| **TXT** | Quick sharing, copy/paste | Small | High |
| **Markdown** | Documentation, wikis | Small | Very High |
| **JSON** | Data analysis, APIs | Medium | Low (technical) |

## Format Details

### Plain Text (TXT)

**Use when**: You need simple, universally compatible text

**Features**:
- Chronological transcript (oldest → newest)
- Speaker labels (Input/Output)
- Timestamps in HH:MM:SS format
- Plain text header with metadata

**Example Output**:
```
Trispr Flow - Transcript Export
Date: 2026-02-14T10:30:00.000Z
Entries: 42

---

[10:30:15] Input: Hello, this is a test recording.
[10:30:22] Output: I hear you loud and clear.
[10:30:28] Input: Great, let's start the meeting.
```

**Ideal For**:
- Email attachments
- Chat messages (Slack, Teams, Discord)
- Simple archival
- Quick copy/paste workflows

---

### Markdown (MD)

**Use when**: You want formatted, readable documentation

**Features**:
- Organized into Input/Output sections
- Markdown formatting (headers, lists, bold)
- Timestamps as bullet points
- Export metadata in bold
- Footer attribution

**Example Output**:
```markdown
# Transcript Export

**Date**: 2026-02-14T10:30:00.000Z
**Total Entries**: 42

---

## Input Transcription

- **10:30:15**: Hello, this is a test recording.
- **10:30:28**: Great, let's start the meeting.

## Output Transcription

- **10:30:22** (Output): I hear you loud and clear.

---
*Generated by Trispr Flow*
```

**Ideal For**:
- Notion, Obsidian, or other note-taking apps
- GitHub/GitLab documentation
- Meeting minutes
- Lecture notes
- Personal knowledge bases

---

### JSON

**Use when**: You need structured data for programmatic access

**Features**:
- Machine-readable format
- Full metadata (export date, format version, entry count)
- ISO 8601 timestamps
- Per-entry source attribution
- Schema versioning (v1.0)

**Example Output**:
```json
{
  "export_date": "2026-02-14T10:30:00.000Z",
  "format_version": "1.0",
  "entry_count": 42,
  "entries": [
    {
      "id": "entry-1708000015000",
      "timestamp_ms": 1708000015000,
      "timestamp": "2026-02-14T10:30:15.000Z",
      "source": "mic",
      "text": "Hello, this is a test recording."
    },
    {
      "id": "entry-1708000022000",
      "timestamp_ms": 1708000022000,
      "timestamp": "2026-02-14T10:30:22.000Z",
      "source": "output",
      "text": "I hear you loud and clear."
    }
  ]
}
```

**Ideal For**:
- API integrations
- Data analysis (sentiment, keywords, etc.)
- Import to other transcription tools (Descript, Otter.ai)
- Custom processing scripts
- Machine learning datasets

## Tips & Tricks

### Selective Export

**Current behavior**: Exports all transcript history (Input + Output combined)

**Workaround for selective export**:
1. Switch to the tab you want (Input/Output/Conv)
2. Use the **Copy** button to copy to clipboard
3. Paste into your preferred editor
4. Save manually

**Future feature (v0.5.0+)**: Selective export per tab

---

### Large Transcripts

**Performance**:
- TXT/MD exports are fast even with 1000+ entries
- JSON exports may take a few seconds for very large datasets

**Size estimates**:
- 100 entries ≈ 10-20 KB (TXT/MD)
- 100 entries ≈ 30-50 KB (JSON)

**Recommendations**:
- For continuous recording sessions (hours-long), consider periodic exports
- JSON is best for archival of large datasets

---

### File Naming

**Default pattern**: `transcript-{YYYY-MM-DD}T{HH-MM-SS}.{ext}`

**Examples**:
- `transcript-2026-02-14T10-30-00.txt`
- `transcript-2026-02-14T10-30-00.md`
- `transcript-2026-02-14T10-30-00.json`

**Tip**: Rename files before saving if you want custom names

## Advanced Usage

### JSON Schema

For developers integrating with Trispr Flow exports, see [EXPORT_SCHEMA.md](EXPORT_SCHEMA.md) for detailed field specifications.

**Key fields**:
- `export_date`: ISO 8601 timestamp
- `format_version`: Schema version (currently "1.0")
- `entry_count`: Total entries
- `entries[]`: Array of transcript entries

**Entry fields**:
- `id`: Unique identifier
- `timestamp_ms`: Unix milliseconds
- `timestamp`: ISO 8601 string
- `source`: `mic`, `output`, or `system`
- `text`: Transcribed text

## Troubleshooting

### Export button is grayed out

**Cause**: No transcript entries available
**Solution**: Record some audio first (Input or Output)

### File dialog doesn't open

**Cause**: Possible permission issue
**Solution**: Run Trispr Flow as administrator (Windows) or check file permissions

### Export fails with error

**Check**:
1. Disk space available
2. Write permissions to chosen directory
3. Valid file path (no special characters)

**If error persists**: Check the app logs and report to [GitHub Issues](https://github.com/YOUR_REPO/issues)

### Exported file is empty

**Cause**: Export filter may be active
**Solution**: Clear search filter, ensure you're viewing the correct tab (Input/Output/Conv)

## Frequently Asked Questions

### Can I export only Input or only Output transcripts?

**Current version**: Use the Copy button on the specific tab, then save manually
**Future version (v0.5.0+)**: Direct export per tab planned

### Can I customize the export format?

**Current version**: No, formats are fixed
**Future version**: Custom export templates planned (post-v0.5.0)

### Does export include audio files?

**No**: Trispr Flow currently exports text only. Audio files are not included.
**Future**: Audio bundling with JSON exports may be added in v0.6.0+

### Can I automate exports?

**Current version**: No automation available
**Future (v0.5.0)**: Live transcript dump (auto-save during recording)

### What character encoding is used?

**All formats**: UTF-8 encoding
**Line endings**: LF (`\n`) preferred, CRLF (`\r\n`) supported
