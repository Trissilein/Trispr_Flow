# v0.8.0 Plan: Module Platform + GDD Automation + Confluence Cloud

Last updated: 2026-03-06
Status: Completed (hardening gate closed)

## Objective

Deliver a managed module platform and ship the first production GDD workflow that turns transcript ranges into strict Game Design Documents and publishes them to Confluence Cloud with resilient queue fallback.

## Milestone Definition of Done

1. Modules can be listed, enabled, disabled, health-checked, and update-checked.
2. GDD module can produce a strict draft from transcript input with preset recognition and override support.
3. User can review draft and publish to Confluence Cloud (create/update lifecycle).
4. One-click publish enforces confidence gating and explicit confirmation fallback on low-confidence routing.
5. Transient publish failures are persisted in a local retry queue with Confluence-compatible fallback bundle artifacts.

## Implementation Status Snapshot

1. Managed module runtime is implemented (registry, lifecycle, permissions, health/update).
2. Modules tab is implemented and wired (including Analyse -> module launcher routing).
3. GDD draft generation pipeline is implemented with preset recognition, clone presets, validation, markdown/storage rendering.
4. Confluence connector is implemented with target suggestion and create/update publish lifecycle.
5. OAuth exchange + refresh and workspace persistence (`oauth_cloud_id`) are implemented.
6. GDD modal workflow is implemented (detect -> generate -> validate -> suggest target -> publish).
7. Template source ingestion is implemented (Confluence URL + PDF/DOCX/TXT/MD) as optional guidance.
8. One-click policy gate is implemented in UI with configurable confidence threshold.
9. Queue/retry fallback commands are implemented and integrated in the modal queue panel.

## Architecture Snapshot

### Core module runtime

- Rust namespace: `src-tauri/src/modules/`
- Components:
  - `registry.rs`: bundled module manifests and descriptor projection
  - `permissions.rs`: consent checks and grant persistence
  - `lifecycle.rs`: enable/disable lifecycle orchestration
  - `health.rs`: health and update signals
- States: `not_installed`, `installed`, `enabled`, `active`, `error`

### GDD module

- Rust namespace: `src-tauri/src/gdd/`
- Components:
  - `presets.rs`: universal strict preset + clone expansion
  - `recognition.rs`: heuristic preset suggestion with confidence output
  - `extraction.rs`: chunked evidence extraction
  - `synthesis.rs`: section synthesis with `TBD` fallback
  - `validation.rs`: strict draft validation
  - `render_storage.rs`: markdown + Confluence storage rendering
  - `confluence.rs`: Cloud connector workflows

### Confluence secrets

- Rust namespace: `src-tauri/src/confluence/keyring.rs`
- Keyring-first storage with file fallback.

## Risks and Mitigations

1. Confluence auth complexity
   - Mitigation: dual auth tracks; API-token path available immediately.
2. Token budget for long transcripts
   - Mitigation: chunked extraction + synthesis architecture.
3. Module instability affecting core app
   - Mitigation: strict module isolation and non-fatal error state transitions.
