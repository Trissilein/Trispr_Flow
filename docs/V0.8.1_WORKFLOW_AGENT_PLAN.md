# v0.8.1 Plan: Workflow-Agent Voice Automation

Last updated: 2026-03-05  
Status: In execution

## Objective

Deliver a production-safe `workflow_agent` module that turns wakeword-triggered transcript commands into a confirm-gated GDD flow (`detect -> resolve session -> plan -> execute -> publish/queue`).

## Scope

1. Wakeword + keyword command detection.
2. Intent parse API (`gdd_generate_publish` in V1).
3. Session search/scoring over transcript history.
4. Candidate confirm and explicit target-language selection.
5. Plan build and execute commands using existing GDD/Confluence queue pipeline.

## Out of Scope

1. Generic multi-tool agent router.
2. No-confirm autonomous execution.
3. Cloud-only agent runtime.
4. External plugin SDK.

## Architecture

1. GDD and Confluence integration are core-always-on.
2. Workflow automation remains optional via `workflow_agent`.
3. Agent uses dedicated command channel `transcription:raw-result` (pre-filter).
4. Legacy activation words stay as transcript drop filter only.
5. Execution requires explicit user confirm.

## API Surface

### Commands

1. `agent_list_supported_actions`
2. `agent_parse_command`
3. `search_transcript_sessions`
4. `agent_build_execution_plan`
5. `agent_execute_gdd_plan`

### Events

1. `agent:command-detected`
2. `agent:plan-ready`
3. `agent:execution-progress`
4. `agent:execution-finished`
5. `agent:execution-failed`

### Settings

`settings.workflow_agent`:

1. `enabled`
2. `wakewords`
3. `intent_keywords`
4. `model`
5. `temperature`
6. `max_tokens`
7. `session_gap_minutes`
8. `max_candidates`

## UX

Modules tab includes Workflow Agent Console:

1. Test command input.
2. Parse command.
3. Session candidate list.
4. Target language selector.
5. Build plan and confirm execute.
6. Execution log.

## Verification

1. Wakeword command emits parse result.
2. Session search handles `today / yesterday / vorgestern / vorhin`.
3. Execute path returns draft-only when publish is off.
4. Publish failures queue correctly for transient Confluence errors.
5. Regression: `cargo check`, `npm run build`, `npm test`.
