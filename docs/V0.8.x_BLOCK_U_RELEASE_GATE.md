# Block U Release Gate + Evidence Packet

Last updated: 2026-03-29

## Scope

This packet defines the executable release-gate flow for Block U (`v0.8.x`) and links all required evidence artifacts.

- `U1` is implemented as assistant UX hardening in the workflow-agent panel.
- `U4` is implemented as gate automation + evidence report generation.
- `U2/U3` soak evidence is intentionally deferred to the end and attached later.

## Automated Gate Runner

Command:

```bash
npm run qa:assistant
```

What it runs:

1. `npm run build`
2. `npm test`
3. `cargo test --manifest-path src-tauri/Cargo.toml --lib`

Output artifacts:

- `docs/reports/block_u_release_gate.latest.json`
- `docs/reports/block_u_release_gate.latest.md`
- timestamped variants in `docs/reports/`

## Optional Strict Modes

Strict benchmark linkage:

```bash
npm run qa:assistant -- --strict-benchmark
```

Skip Rust lib tests (environment workaround):

```bash
npm run qa:assistant -- --skip-rust-lib-tests
```

Require soak evidence attachments:

```bash
npm run qa:assistant -- --require-soak --soak8 docs/reports/assistant_soak_8h.json --soak24 docs/reports/assistant_soak_24h.json
```

Combined strict mode (final sign-off run):

```bash
npm run qa:assistant -- --strict-benchmark --require-soak --soak8 docs/reports/assistant_soak_8h.json --soak24 docs/reports/assistant_soak_24h.json
```

Note: `--skip-rust-lib-tests` must not be used for final sign-off.

## Evidence Sources

Benchmark-linked diagnostics (pre-existing):

- `bench/results/latest.json` (latency benchmark)
- `bench/results/tts.latest.json` (TTS benchmark)

Block U gate report (new):

- `docs/reports/block_u_release_gate.latest.json`
- `docs/reports/block_u_release_gate.latest.md`

Deferred soak attachments (to be added at end):

- `docs/reports/assistant_soak_8h.json`
- `docs/reports/assistant_soak_24h.json`

## Exit Criteria

Pre-soak (`U1 + U4 prep`) is considered complete when:

1. `npm run qa:assistant` passes.
2. Gate report artifacts are generated and committed.
3. Benchmark diagnostics are linked in report (strict optional before final sign-off).

Final release sign-off requires:

1. `U2` 8h soak evidence attached.
2. `U3` 24h soak evidence attached.
3. strict run passes (`--strict-benchmark --require-soak`).
